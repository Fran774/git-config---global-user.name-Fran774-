1. Što je Visual Studio Code i za što se najčešće koristi?
Visual Studio Code je besplatan uređivač izvornog koda koji je razvio Microsoft. VS Code se koristi za razvoj softvera i web aplikacija
2. Kako se zove panel u kojem vidiš sve datoteke i mape svog projekta?
Panel u kojem vidiš sve datoteke i mape svog projekta zove se Explorer.
3. Kako možeš instalirati ekstenzije u VS Codeu i čemu one služe? 
Možemo instalirati putem panela "Extensions", korištenjem naredbene palete i instalacija s Marketplacea. Ekstenzije služe za prilagodbu VS Codea specifičnim potrebam projekta ili programskog jezika. 
4. Kako otvoriti postojeći projekt ili mapu u VS Code-u? 
Projekt ili mapu možemo otvoriti korištenjem izbornika "File", korištenjem naredbene palete, povlačenjem i ispuštanjem, korištenjem naredbenog retka.
5. Što se sve može raditi u Exploreru?
U Exploreru upravljanje i navigaciju doatoteka i mapa, možemo pregeldavati strukture, otvarati datoteke, stvarati nove datoteke.
6. Kako možeš brzo pronaći određenu datoteku u projektu?
Datoteku možemo pronaći korištenjem "Go to File", naredbene palete, pretraživanjem unutar explorera, "Search Panela".
7. Kako možeš istovremeno gledati dvije različite datoteke u editiru?
Istovremeno možemo gledati dvije različite datoteke korištenjem Split Editora?
8. Kako pokrećeš automatsko formatiranje koda u VS Code-u?
Automatsko formatiranje pokrećemo tako da označimo kućicu pored "Editor: Format on save".
9. Što je version control i zašto se koristi? 
Version control je sustav koji bilježi promjene na datoteci ili skupu datoteka, koristi se za: praćenje povijesti, učinkovitu suradnju tima, dokumentacija promjena.
10. Kako se u VS Code-u otvara panel za Git (Source Control)?
Nalazi se nalijevoj bočnoj traci, ikona izgleda kao malo stablo,.
11. Što znače commit, push, pull?
Commit je spremanje trenutnog stanja promjena koje smo napravili na lokalnom repozitoriju, push je slanje lokalnih commitiva na udaljeni repozitorij, pull je preuzimanje promjena s udaljenog repozitorija.
12. Čemu služi commit poruka i kako treba izgledati?
Commit poruka je karatak opis koji se dodaje svakom commitu.
13. Kako možeš vidjeti koje si promjene napravio u podjedinoj datoteci?
Kroz source control panel i timeline panel.
14. Kreiraš novu datoteku. Koji su koraci da se promjena pošalje na GitHub?
U gornjem desnom kutu kliknemo "Push".
15. Kako se vraćaš na stariju verziju datoteke ako si pogriješio?
U Source Controlu odbacimo promjene.
16. Na koji način možeš provjeriti status projekta i koje datoteke su izmijenje?
Kroz sučelje i kroz terminal.
